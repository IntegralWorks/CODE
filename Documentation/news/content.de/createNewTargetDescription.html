Arbeiten mit Target Beschreibungsdateien

CODE ermöglicht die Erstellung von Beschreibungsdateien (COBOL build file description) für die Erstellung unserer Kompilate. Die Dateien enden mit cbf für COBOL build file. Wir können in einem Projekt beliebig viele dieser Dateien hinzufügen. Die Beschreibungsdateien entsprechen vom Aufbau unseren COBOL Dateien und enthalten in Spalte 7 entsprechende Annotationen zu der Zeile.

AnnotationBedeutung
*Kommentarzeile
T=EXECUTABLE erzeugt ein ausführbares Programm.
Die nachfolgenden Zeilen beschreiben dieses Programm.
T=LIBRARY erzeugt ein Bibliothek.
Die nachfolgenden Zeilen beschreiben diese Bibliothek.
NName des Kompilats (Programm oder Bibliothek).
CInhalt der dem Kompilat hinzugefügt werden soll.
Dahinter ist der voll qualifizierte Dateiname zu schreiben.
<SPACE>Alternativschreibweise für C

...oder für uns (aus der Dokumentation von BuildTargetCpyReader): 

01 COBOL-FORMAT                  PIC X(80).
  05 SEQUENCE-NUMBER               PIC X(6).
  05 INDICATOR-AREA                PIC X.
    88 INDICATOR-COMMENT             VALUE '*'.
    88 TARGET-NAME                   VALUE 'N'.
    88 TARGET-TYPE                   VALUE 'T'.
    88 TARGET-CONTENT                VALUE 'C', ' '.
  05 PROGRAM-TEXT                  PIC X(65).
  05 REFERENCE-AREA                PIC X(8).
01 FILLER REDEFINES COBOL-FORMAT.
  05 SEQUENCE-NUMBER               PIC X(6).
  05 ACCEPTED-TARGET-TYPES         PIC X(74).
    88 TARGET-TYPE-EXECUTABLE        VALUE 'T=EXECUTABLE'.
    88 TARGET-TYPE=LIBRARY           VALUE 'T=LIBRARY'.
